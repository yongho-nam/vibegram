# 프론트엔드 개발 명세서 (Instagram 클론)

## 1. 개요

- **목표**: 모바일·데스크톱 우선(UI는 주로 반응형 웹으로 모바일 앱 플로우에 가깝게 구현) 인스타그램 핵심 UX를 재현한다.
- **기술 스택**: React 18+, TypeScript 권장, 빌드 도구(Vite 권장), 클라이언트 라우팅(React Router v6+).

## 2. 기능 범위(완전한 클론 기준 모듈)

| 영역 | 기능 |
|------|------|
| 인증 | 회원가입, 로그인, 로그아웃, 비밀번호 변경(선택), 세션 만료 처리 |
| 피드 | 홈 무한 스크롤, 스토리 링, 추천/팔로잉 피드 토글(선택) |
| 게시물 | 생성(이미지 다중 업로드, 캡션, 위치·태그는 선택), 상세, 수정·삭제(본인) |
| 상호작용 | 좋아요(게시물·댓글), 댓글 CRUD, 북마크(저장) |
| 프로필 | 본인/타인 프로필, 그리드, 팔로워·팔로잉 수, 팔로우/언팔로우 |
| 검색 | 사용자·해시태그 검색, 탐색 그리드(선택) |
| 스토리 | 24시간 만료 콘텐츠 표시, 뷰어, 본인 스토리 업로드(선택) |
| DM | 대화 목록, 1:1 메시지, 실시간은 WebSocket 또는 폴링으로 백엔드와 합의 |
| 알림 | 좋아요·댓글·팔로우·멘션 등 알림 목록, 읽음 처리 |
| 설정 | 프로필 편집(이름, 바이오, 아바타), 비공개 계정 토글(선택), 계정 삭제(선택) |

백엔드에서 미구현인 기능은 UI를 비활성화하거나 “준비 중”으로 처리할 수 있으나, 명세상으로는 위 범위를 목표로 한다.

## 3. 화면(라우트) 구조

- `/login`, `/signup`
- `/` — 홈 피드
- `/explore` — 탐색(선택)
- `/search` — 검색
- `/create` — 게시물 작성
- `/p/:postId` — 게시물 상세(딥링크)
- `/:username` — 프로필
- `/:username/followers`, `/:username/following`
- `/direct` — DM 목록
- `/direct/t/:threadId` — DM 스레드
- `/accounts/edit` — 프로필 편집
- `/notifications` — 알림
- `*` — 404

## 4. 상태·데이터

- **서버 상태**: React Query(TanStack Query) 또는 SWR 권장 — 캐시 무효화(좋아요·댓글 후 `invalidateQueries`).
- **인증 상태**: Context 또는 경량 전역 스토어 + `access_token`(또는 HttpOnly 쿠키 사용 시 `credentials: 'include'`).
- **낙관적 업데이트**: 좋아요·팔로우에 적용하여 체감 속도 개선, 실패 시 롤백.

## 5. API 연동 규칙

- **Base URL**: 환경변수 `VITE_API_BASE_URL`(또는 동등)로 분리.
- **인증 헤더**: Bearer JWT(백엔드와 동일 스키마).
- **에러**: 401 시 로그인 화면으로 리다이렉트(선택: refresh 토큰 플로우는 백엔드와 합의).
- **페이지네이션**: `cursor` 또는 `offset/limit` — 백엔드 `backend.md`와 필드명 통일.
- **미디어 업로드**: `multipart/form-data`로 이미지 배열 전송, 진행률 표시(선택).

## 6. UI·UX 가이드

- 하단(모바일)·상단(데스크톱) 내비게이션으로 핵심 이동 경로 고정.
- 이미지 비율 1:1 그리드, 피드는 세로 카드, 스켈레톤 로딩.
- 접근성: 버튼에 `aria-label`, 모달 포커스 트랩, 키보드 닫기.
- 다국어는 선택 사항(초기 한국어 또는 영어 단일 언어 가능).

## 7. 보안·클라이언트 측 주의

- 토큰을 `localStorage`에 둘 경우 XSS 위험을 인지하고, 가능하면 HttpOnly 쿠키 + CSRF 전략을 백엔드와 맞춘다.
- 사용자 입력(캡션·댓글)은 표시 시 이스케이프/리치 텍스트 정책을 백엔드와 일치시킨다.

## 8. 테스트·품질

- 핵심 플로우 E2E(Playwright 등) 1~2개: 로그인 → 피드 → 좋아요.
- 단위 테스트: 유틸·날짜 포맷·API 클라이언트 래퍼(선택).
